Logic after every completed ride/trip:
1. Find all active bonus targets where now() BETWEEN startsAt AND endsAt
2. For each target that matches the service type:
   - Upsert `driverBonusProgress` for this driver + target
   - Increment `currentValue` by 1 (ride_count) or by finalPrice (earnings_amount)
   - If `currentValue >= targetValue` AND `isCompleted = false`:
     - Set `isCompleted = true`, `completedAt = now()`
     - Add `bonusAmount` to driver's wallet
     - Insert wallet transaction: type `bonus`, description "Bonus: {target.name}"
     - Insert driver_earnings row: type `milestone_bonus`
     - Notify driver: "🎉 You completed the bonus target and earned X EGP!"
     - Emit socket event `driver:bonus:completed` to `driver:{userId}` room

Admin endpoints:
- `GET /admin/bonus-targets` — list all targets with stats 
  (how many drivers enrolled, how many completed)
- `POST /admin/bonus-targets` — create new target
- `PATCH /admin/bonus-targets/:id` — update (can deactivate mid-run)
- `DELETE /admin/bonus-targets/:id` — soft delete
- `GET /admin/bonus-targets/:id/progress` — see per-driver progress for a target
- `GET /admin/drivers/:id/bonus-progress` — see all bonus progress for one driver

Driver endpoints (authenticated driver):
- `GET /driver/bonus-targets` — see active targets with my current progress
  Returns: target details + currentValue + targetValue + isCompleted + bonusAmount
