تقرير: بيانات الـ Departure Time في صفحة حجز الـ Lines
ملخص سريع
الـ departureTime في صفحة الحجز يأتي بالكامل من الـ Backend — لا يوجد أي حساب أو توليد من الـ Client.

١. من أين تجيب الداتا؟
API Call واحد فقط يغذّي الـ departure time في صفحة الحجز:

GET /shuttle/lines/{routeId}/available-weeks

هذا الـ endpoint يرجع البيانات بالشكل ده:

{
  "weeks": [
    {
      "weekStart": "2026-06-21",
      "weekEnd": "2026-06-25",
      "slots": [
        {
          "id": 1,
          "departureTime": "07:30",   ← ده هو اللي بيتعرض
          "totalSeats": 14,
          "availableSeats": 10,
          "isBooked": false,
          "isTaken": false
        }
      ]
    }
  ]
}

٢. إزاي بيتعرض؟
الـ flow هو كالتالي:

المستخدم يختار Route
        ↓
useQuery(['shuttle-available-weeks', routeId])
        ↓
GET /shuttle/lines/{routeId}/available-weeks
        ↓
availableWeeksData.weeks[i].slots[j].departureTime
        ↓
currentSlots = selectedWeek.slots
        ↓
يتعرض على الـ UI كـ slot card

الـ departureTime هو string بصيغة "HH:MM" (مثلاً "07:30")
لا يوجد أي تحويل أو formatting — يتعرض كما هو من الـ server
كل أسبوع (BackendWeek) جواه قائمة slots وكل slot عنده departureTime خاص بيه
٣. باقي الأماكن اللي بتعرض departure time
المكان	مصدر الداتا	الشكل
Booking modal (صفحة الحجز)	availableWeeks API ← slot.departureTime	"HH:MM" كما هو من الـ server
Confirm alert	نفس الـ selectedSlot.departureTime	"HH:MM"
My Bookings card	myBookings API ← GET /shuttle/route-bookings ← booking.departureTime أو booking.timeSlot.departureTime	"HH:MM"
Cancel/Renew dialogs	booking.departureTime من الـ ShuttleBooking object	"HH:MM"
Lines index (الـ Home)	ShuttleLine.departure ← formatTime(trip.departureTime) من GET /trips ← يحوّله من ISO إلى HH:MM	
٤. ملاحظة مهمة — فرق بين context layers
هناك نوعان من الـ departure time في الكود:

الـ Booking Layer (صفحة الـ Lines):

مصدره: available-weeks endpoint
الصيغة: "HH:MM" string ثابتة من الـ server
لا يوجد أي تعديل من الـ client
الـ Trip Execution Layer (الـ Home/Active trip):

مصدره: trips endpoint → trip.departureTime كـ ISO date
يمر بـ formatTime() في shuttleContext.tsx اللي تحوّله لـ HH:MM باستخدام toLocaleTimeString
الخلاصة: الـ departure time في صفحة حجز الـ Lines هو قيمة "HH:MM" بتيجي مباشرة من الـ Backend عبر endpoint الـ available-weeks، ومش بيتعمله أي حسابات أو تعديلات على الـ client side.
