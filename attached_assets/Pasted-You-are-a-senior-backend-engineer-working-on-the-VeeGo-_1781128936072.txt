You are a senior backend engineer working on the VeeGo shuttle system.
Phase 1 and Phase 2 fixes are already done. I need you to implement one specific endpoint.
Context
In Phase 2, we added full-week coverage validation to POST /shuttle/route-bookings.
A driver cannot book a slot unless all 5 working days (Sunday to Thursday) have trips in that week.
The problem now: The driver app currently shows all slots to the driver, including slots that would fail this validation. The driver taps a slot, tries to book, and gets an error. Bad experience.
The fix: A new endpoint that only returns slots that pass the full-week coverage check, so the driver app only shows bookable slots.
What to Build
New Endpoint: GET /shuttle/available-slots
Who calls it: Driver app (authenticated driver only)
Query params:
routeId (required) — the route the driver is browsing
weekStart (required) — ISO date string for the Sunday of the target week (Cairo time)
What it returns:
A list of time slots for that route and week that:
Have trips on ALL 5 working days (Sunday to Thursday)
Are NOT already booked by another driver for that week
Are still in a bookable trip status (not cancelled, not completed)
Response shape:
Json
Rules:
Reuse the same Cairo day-of-week logic from Fix 4 in Phase 2.
Use VEHICLE_CAPACITY and VEHICLE_MIN_THRESHOLD from the schema for totalSeats and minRequired.
If weekStart is not a valid ISO date or not a Sunday in Cairo time, return HTTP 400 with a clear message.
Requires authenticate + requireRole("driver") middleware.
