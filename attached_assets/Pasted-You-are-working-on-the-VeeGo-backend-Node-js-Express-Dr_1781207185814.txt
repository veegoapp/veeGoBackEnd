## FIX 2 ‚Äî Car Service Category System (Economy / Economy Plus / Comfort)

**Problem:**
The Car service currently uses a generic `vehicleType: "car"` with no category 
distinction. The spec requires 3 categories based on vehicle manufacture year,
with special rules for Chinese brands.

**Category Rules (default ‚Äî must be configurable from admin):**

Default year ranges:
- Economy:       year 2000 ‚Üí 2007
- Economy Plus:  year 2008 ‚Üí 2019
- Comfort:       year 2020 ‚Üí current year

Chinese brands exception:
- Economy Plus starts from 2014 (not 2008) for Chinese brands
- Chinese brand is determined by `isChinese = true` in `vehicleBrands` table
  (already built in FIX 1 ‚Äî do not create a separate chineseBrands table)

Cross-category rules:
- Comfort driver ‚Üí can accept Economy, Economy Plus, Comfort rides
- Economy Plus driver ‚Üí can accept Economy, Economy Plus rides
- Economy driver ‚Üí can ONLY accept Economy rides

**What to do:**

1. Create table `carCategories`:
   - `id` (serial, PK)
   - `name` (text) ‚Äî "Economy", "Economy Plus", "Comfort"
   - `slug` (text, unique) ‚Äî "economy", "economy_plus", "comfort"
   - `minYear` (integer)
   - `maxYear` (integer, nullable ‚Äî null = current year)
   - `baseFare` (numeric 10,2)
   - `perKmRate` (numeric 10,2)
   - `perMinuteRate` (numeric 10,2)
   - `minimumFare` (numeric 10,2)
   - `isActive` (boolean, default true)
   - `sortOrder` (integer)
   - `createdAt` (timestamp)

2. Seed 3 categories:
   - Economy:      minYear=2000, maxYear=2007, baseFare=8,  perKm=4,   perMin=0.5, minFare=15, sortOrder=1
   - Economy Plus: minYear=2008, maxYear=2019, baseFare=12, perKm=5.5, perMin=0.7, minFare=20, sortOrder=2
   - Comfort:      minYear=2020, maxYear=null,  baseFare=18, perKm=7,   perMin=1.0, minFare=30, sortOrder=3

3. Add column `categoryId` (FK ‚Üí carCategories, nullable) to `vehicles` table

4. Create helper function `resolveCarCategory(brandId: number, year: number)`:
   - Lookup brand in `vehicleBrands` to get `isChinese`
   - If isChinese = true:
     - year < 2014 ‚Üí Economy
     - year 2014-2019 ‚Üí Economy Plus
     - year >= 2020 ‚Üí Comfort
   - If isChinese = false: use standard ranges from `carCategories` table
   - If year < 2000 ‚Üí throw error "Vehicle year not accepted"
   - Returns `{ categoryId, slug, name }`

5. Auto-resolve and save categoryId:
   - In `POST /api/vehicles`: call resolveCarCategory(brandId, year)
   - In `PATCH /api/vehicles/:id`: re-resolve if brandId or year changed
   - In driver registration vehicle step: resolve and save

6. Update `POST /api/rides/request`:
   - Accept optional `categorySlug`
   - If not provided ‚Üí default to "economy"
   - Dispatch based on cross-category rules above
   - Save `requestedCategory` on ride (add nullable text column to rides table)

7. Update `POST /api/rides/estimate`:
   - If no categorySlug ‚Üí return all 3 active categories with pricing as array
   - If categorySlug ‚Üí return that category pricing only

8. Admin endpoints:
   - `GET /api/admin/car-categories` ‚Äî list all
   - `PATCH /api/admin/car-categories/:id` ‚Äî update any field

9. Public endpoint:
   - `GET /api/car/categories` ‚Äî active categories sorted by sortOrder

**Test:**
- Vehicle: brandId=Toyota(non-Chinese), year=2005 ‚Üí Economy
- Vehicle: brandId=Toyota, year=2015 ‚Üí Economy Plus
- Vehicle: brandId=Toyota, year=2022 ‚Üí Comfort
- Vehicle: brandId=BYD(isChinese=true), year=2012 ‚Üí Economy
- Vehicle: brandId=BYD, year=2016 ‚Üí Economy Plus
- `POST /api/rides/estimate` no slug ‚Üí returns 3 categories
- `POST /api/rides/request` slug="economy" ‚Üí dispatches Economy+EcoPlus+Comfort drivers
- `POST /api/rides/request` slug="comfort" ‚Üí Comfort drivers only
