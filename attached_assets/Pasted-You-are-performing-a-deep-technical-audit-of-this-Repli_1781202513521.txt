You are performing a deep technical audit of this Replit project.

This is the Backend Server + Admin Dashboard for a ride-hailing app called VeeGo — similar to Uber — with 4 services: Shuttle (Shatel), Car, Scooter, and Delivery. The backend is built with Node.js and controls everything: the Rider app, Driver app, and Admin Dashboard.

Your job is to produce a comprehensive audit report as a `.md` file named `audit_backend.md`.

---

## AUDIT SCOPE

### 1. PROJECT STRUCTURE
- List every folder and file with a one-line description of its purpose
- Identify the main entry point
- Identify any dead files or unused modules

### 2. DATABASE
- List all database models/schemas with their fields
- Identify relationships between models
- Flag any missing models that are referenced but not defined
- Flag any models that exist but seem unused
- Note: which database is being used (MongoDB, PostgreSQL, etc.) and how it's connected

### 3. API ENDPOINTS — For EVERY endpoint list:
- HTTP Method + full path
- What it does
- Authentication/authorization required (yes/no/type)
- Input parameters (body, query, params)
- What it returns
- Status: ✅ Fully working | ⚠️ Partially working / needs improvement | ❌ Broken / mock data / not connected | 🗑️ Dead / unused
- Notes on issues found

### 4. SOCKET / REAL-TIME
- List all socket events (emit and on) with descriptions
- Which events are connected end-to-end vs. mocked or broken
- Missing socket events that the app logic requires

### 5. AUTHENTICATION & SECURITY
- How is auth implemented (JWT, sessions, OTP, etc.)?
- Are tokens validated properly on protected routes?
- Any missing auth guards on sensitive routes?
- OTP flow — is it real or mocked?

### 6. SERVICES IMPLEMENTATION STATUS
For each of the 4 services (Shuttle, Car, Scooter, Delivery), report:
- What is built
- What is mocked or hardcoded
- What is missing entirely
- Is the service toggleable from the admin dashboard?

### 7. ADMIN DASHBOARD
- List all admin dashboard pages/sections
- What data does each page display and from where?
- Which controls/actions actually work vs. are UI-only
- Missing admin features

### 8. VEHICLE & PRICING SYSTEM
- Is the vehicle brand/model/year system implemented?
- Is the category system for Car service implemented (Economy / Economy Plus / Comfort by year ranges)?
- Is pricing configurable per service and per category from the admin?
- Any hardcoded prices?

### 9. SHUTTLE (SHATEL) SPECIFIC
- Routes (Lines) and Stations system — implemented?
- Trip scheduling (dates + times) — implemented?
- Minimum passenger logic (7 for HiAce, 14 for Mini Bus) — Pending → Active auto-flip — implemented?
- Trip cancellation with 10-hour rule — implemented?
- Driver weekly booking system — implemented?
- Driver conflict detection (time + geography) — implemented?
- Driver trip transfer to another driver — implemented?

### 10. NOTIFICATIONS
- What notification system is used (push, in-app, SMS)?
- Which notifications are implemented vs. missing?

### 11. DOCUMENT UPLOAD & DRIVER VERIFICATION
- Is document upload implemented (license, vehicle registration, personal photo, vehicle photo, criminal record)?
- Is admin review + account activation flow implemented?
- Is the 30-trip threshold for criminal record implemented?
- Is multi-vehicle/multi-account alert implemented?

### 12. WALLET & PAYMENTS
- What payment methods are implemented?
- Is cash-only mode implemented and controllable from admin?
- Is wallet balance, transaction history, and card management built?

### 13. DEAD CODE & UNNECESSARY FEATURES
- List any files, routes, models, or features that exist in the code but serve no purpose or conflict with the project spec

---

## OUTPUT FORMAT

Generate a single file: `audit_backend.md`

Use this structure:
- Use `##` for each section
- Use tables where listing endpoints, models, or events
- Use status emojis: ✅ ⚠️ ❌ 🗑️
- End with a prioritized summary table:

| # | Item | Status | Priority |
|---|------|--------|----------|
| 1 | ... | ❌ Missing | 🔴 High |

Be exhaustive. Do not summarize or skip details. If you are unsure about something, flag it with ❓.
