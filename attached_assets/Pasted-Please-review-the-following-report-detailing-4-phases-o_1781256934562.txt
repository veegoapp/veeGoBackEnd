# BACKEND IMPLEMENTED MANIFEST FOR FRONTEND AUDIT

### 1. Wallet Manipulation Prevention & Settings (Phase 1)
- **Backend Capability:** Endpoint `PATCH /api/admin/settings/wallet-limits` updates two global keys in the settings table: `wallet_max_topup` and `wallet_daily_topup_limit`. Rejects top-ups if limits are exceeded.
- **Audit Question:** Is there a control panel or input fields in the Admin Dashboard to read and change these two specific wallet limits, or is it hidden/missing?

### 2. Auto Driver Account Activation (Phase 1)
- **Backend Capability:** `PATCH /api/driver-documents/:id` triggers auto-activation of both `drivers.isActive` and `users.isVerified` to `true` when exactly 8 core documents are marked 'approved'.
- **Audit Question:** Does the admin document approval screen clearly inform or visually represent to the admin that approving the 8th document will trigger automatic live activation?

### 3. Auto Refund on Trip Cancellation (Phase 1)
- **Backend Capability:** `PATCH /api/trips/:id/cancel` and `DELETE /api/trips/:id` execute atomic transactions that automatically refund full pricing back to user wallets and push notifications.
- **Audit Question:** When an admin views trips/bookings, is the cancel/delete action wired up to these specific endpoints? Are the refund outcomes visible on the UI?

### 4. Shuttle Vehicle Types & Dynamic Capacity (Phase 2)
- **Backend Capability:** New table `shuttle_vehicle_types` with active CRUD endpoints: `GET/POST /admin/shuttle/vehicle-types` and `GET/PATCH/DELETE /admin/shuttle/vehicle-types/:id` supporting dynamic capacities (e.g., `hiace` with 14 seats, `minibus` with 28 seats).
- **Audit Question:** Do we have an independent management screen or configuration catalog for "Shuttle Vehicle Types" in the dashboard, or are these types unmanageable from the UI?

### 5. Criminal Record Enforcement & Auto-Suspension (Phase 2)
- **Backend Capability:** Automatically suspends drivers with ≥30 completed rides/trips if their criminal record isn't approved (`criminal_record_trip_threshold`). Includes an auto-reactivation hook upon document approval and a manual audit endpoint `POST /admin/drivers/:id/check-criminal-record`.
- **Audit Question:** Is the driver's total trip count vs. threshold visible in their admin profile? Is there a manual "Trigger Criminal Record Check" button hitting the `POST` endpoint? Can admins see if a driver is suspended specifically due to this block?

### 6. Duplicate Driver Fraud Detection (Phase 2)
- **Backend Capability:** Table `driver_duplicate_alerts` logs fraud flags when drivers share a `nationalId`. Includes endpoints `GET /admin/duplicate-alerts` (paginated with `?resolved=true/false` filters) and `PATCH /admin/duplicate-alerts/:id/resolve`.
- **Audit Question:** Is there a "Fraud Alerts" or "Duplicate Driver Alerts" review center/screen in the admin panel to view, filter, and resolve these flags?

### 7. Vehicle Catalog Extensions (Phase 3)
- **Backend Capability:** Tables `vehicle_brands`, `vehicle_models`, and `vehicle_colors`. Includes full admin CRUD under `/admin/vehicle-catalog/brands` and `/admin/vehicle-catalog/models` (Note: Brands/Models UI was addressed in Phase 2 frontend, verify if completely covered).
- **Audit Question:** Is there any UI control to view, add, or manage the `vehicle_colors` table via `GET /vehicle-catalog/colors`?

### 8. Dynamic Settings Unified Engine (Phase 3)
- **Backend Capability:** Endpoints `GET /admin/settings`, `PATCH /admin/settings` (upsert key/value rows), and `GET /admin/settings/:key`. Core seeded keys include: `dispatch_radius_km`, `dispatch_max_radius_km`, `dispatch_offer_timeout_seconds`, `no_show_fee_egp`, `cancellation_grace_hours`.
- **Audit Question:** Is there a centralized "System Global Settings" screen where an admin can adjust these raw dispatching, radius, and timing parameters?

### 9. Driver Incentives, Overrides & Exemptions (Phase 4)
- **Backend Capability:** 
  - Per-Driver Commission Override via `PATCH /api/admin/drivers/:id/commission` (sets specific rate or `null` to clear).
  - Commission Exemption Periods table (`driver_commission_exemptions`) with full CRUD (`GET/POST/PATCH/DELETE` under `/api/admin/commission-exemptions`).
  - Milestone Bonus System (`driver_bonus_targets`) with complete CRUD (`GET/POST/PATCH/DELETE` under `/api/admin/bonus-targets`) and progress tracking screens (`/admin/bonus-targets/:id/progress` and `/admin/drivers/:id/bonus-progress`).
- **Audit Question:** Are these three massive incentive features (Personal Commission Override, Exemption Windows, and Milestone Bonus Targets CRUD/Progress) built into the admin dashboard UI, or are they entirely missing from the frontend?
