=====================================================

GET /api/admin/bookings
Auth: Bearer JWT + admin role

- Passenger name
- Trip details (route, departure time)
- Seat count
- Total price
- Payment status
- Booking status

- View Details (show full booking info in a dialog)
- Cancel Booking → PATCH /api/admin/bookings/:id/cancel
- Refund → POST /api/admin/wallet/refund

RETRY ALL — Job Queue
=====================================================
Add a "Retry All" button to the Queue Monitor card 
that re-enqueues every dead-letter entry in one click.

Endpoint to add:
POST /api/admin/queue/retry-all
Auth: Bearer JWT + admin role
Response: { success: true, retriedCount: number, pendingCount: number }

The button should:
- Show "Retrying..." with spinner while in-flight
- Disable all other retry buttons while running
- Refresh the queue monitor card on success
